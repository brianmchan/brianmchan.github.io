Static vs non-static methods: May 30, 2014

I never truly understood the differences between static and non-static methods. When thinking of non-static methods, I thought about how I created objects in CSE 143 and basically none of the classes I wrote dealt with static methods as opposed to in CSE 142 where basically all of the methods were static. While working on the course scheduler project and reading Core Java, I have begun to understand what it means to be static.

A static method belongs to a class itself while a non-static method belongs to an object. A great example of static methods is the use of the Math class in Java. When you want to use a method in the Math class, say pow, you do not need to initialize a Math object to perform the calculation. Static methods are methods that do not operate on objects.

Math 480B: April 13, 2014

I am finally writing my first blog post. This quarter is my last as an undergraduate at the University of Washington and I want to finish my undergraduate career on a good note.

I only need one more math class to fulfill my degree and I found the perfect class that was being offered in Spring 2014. I also wanted to learn more about Python. I am currently taking Math 480B, which is using Sage, an open source mathematical software using Python. Many people say it's a great class and I hope to learn a lot from the class. I am learning a lot in that class. List comphrension is pretty cool and it's filling in the gaps about Python that I haven't learned before.

Example of list comphrension:

my_data = [n**Integer(2) for n in range(Integer(10))]

The cool thing about list comprehensions is that it is concise and easy to understand and instead of writing out a loop, everything can be written in just one line.

As for Sage, I am liking it so far. It is much better than using MATLAB. Also, Professor Stein creating the SageMathCloud website is a great idea.
